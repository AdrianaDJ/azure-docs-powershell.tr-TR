# Rename-AzContext

## SYNOPSIS
Azure bağlamını yeniden adlandırma.  Varsayılan olarak, Kullanıcı hesabı ve abonelikle adlandırılır.

## INDEKI

### RenameByInputObject (varsayılan)
```
Rename-AzContext -InputObject <PSAzureContext> [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-TargetName] <String> [<CommonParameters>]
```

### RenameByName
```
Rename-AzContext [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-SourceName] <String> [-TargetName] <String>
 [<CommonParameters>]
```

## Tanım
Azure bağlamını yeniden adlandırma.  Varsayılan olarak, Kullanıcı hesabı ve abonelikle adlandırılır.

## ÖRNEKLERDEN

### Adlandırılmış parametreler kullanarak bağlamı yeniden adlandırma
```
PS C:\> Rename-AzContext -SourceName "[user1@contoso.org; 12345-6789-2345-3567890]" -TargetName "Work"
```

' user1@contoso.org ' ' 12345-6789-2345-3567890 ' aboneliği ile ' ' çalışma '  Bu komuttan sonra, ' Select-AzContext çalışması ' kullanarak bağlamı hedefleyebilirsiniz.  Sekme tamamlamayı kullanarak ' SourceName ' değerleri arasında sekme alabilirsiniz.

### Konum parametrelerini kullanarak bağlamı yeniden adlandırma
```
PS C:\> Rename-AzContext "My context" "Work"
```

"My Context" adlı bağlamı "Iş" olarak yeniden adlandırın.  Bu komuttan sonra, Select-AzContext Iş kullanarak bağlam hedefleyebilirsiniz
