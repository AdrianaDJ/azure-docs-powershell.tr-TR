# Enable-AzSqlInstanceAdvancedDataSecurity

## SYNOPSIS
Yönetilen örnekte gelişmiş veri güvenliğini verir.

## INDEKI

```
Enable-AzSqlInstanceAdvancedDataSecurity [-DoNotConfigureVulnerabilityAssessment] [-AsJob]
 [-DeploymentName <String>] [-InputObject <AzureSqlManagedInstanceModel>] -InstanceName <String>
 [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## Tanım
**Enable-Azsqlınstanceadvancevseçdatasecurity** cmdlet 'i yönetilen örnekte gelişmiş veri güvenliğini etkinleştirir. Gelişmiş veri güvenliği, yönetilen örneğiniz için veri sınıflandırması, güvenlik açığı değerlendirmesi ve Gelişmiş tehdit koruması içeren Birleşik bir güvenlik paketidir. (Güvenlik açığı değerlendirmelerinin kaydedilmesi için yeni bir depolama hesabı otomatik olarak oluşturulacaktır. Bu amaçla önceden bir depolama hesabı oluşturulduysa, bunun yerine bu bir hesap kullanılır.

## ÖRNEKLERDEN

### Örnek 1: yönetilen örneği gelişmiş veri güvenliğini etkinleştirme
```powershell
PS C:\>  Enable-AzSqlInstanceAdvancedDataSecurity `
            -ResourceGroupName "ResourceGroup01" `
            -InstanceName "ManagedInstance01" 

ResourceGroupName            : ResourceGroup01
ManagedInstanceName          : ManagedInstance01
IsEnabled                    : True
```

### Örnek 2: sunucu kaynağından yönetilen örnek gelişmiş veri güvenliğini etkinleştirme
```powershell
PS C:\>  Get-AzSqlInstance `
           -ResourceGroupName "ResourceGroup01" `
           -Name "ManagedInstance01" `
           | Enable-AzSqlInstanceAdvancedDataSecurity

ResourceGroupName            : ResourceGroup01
ManagedInstanceName          : ManagedInstance01
IsEnabled                    : True
```

### Örnek 3

Yönetilen örnekte gelişmiş veri güvenliğini verir. oluşturulmuş

```powershell
<!-- Aladdin Generated Example --> 
Enable-AzSqlInstanceAdvancedDataSecurity -DoNotConfigureVulnerabilityAssessment -InstanceName 'ContosoManagedInstanceName' -ResourceGroupName MyResourceGroup
```
