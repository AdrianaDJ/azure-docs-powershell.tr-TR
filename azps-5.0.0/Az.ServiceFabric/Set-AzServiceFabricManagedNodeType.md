## Tanım
Düğüm türü kaynak özelliklerini ayarlar ya da-Reimage parametresi Reimgae işleminde, VM 'leri yeniden yazdırmadan ve geri gelmeleri için yeniden etkinleşmeden önce hizmet yapısı düğümleri devre dışı bırakılır. Bu birincil düğüm türlerinde yapıldığında, aynı anda tüm düğümlerin yeniden görüntüsü olabileceğinden biraz zaman alabilir. Use-Forcereımage, hizmet yapısı düğümleri devre dışı bırakmasa da işlemi zorla, ancak durum bilgisi olan iş yükleri düğümde çalışıyorsa veri kaybına yol açabilir.

## ÖRNEKLERDEN

### Örnek 1
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Set-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName -name $NodeTypeName -InstanceCount 6 -Verbose
```

Düğüm türünün örnek sayımını güncelleyin.

### Örnek 2
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Set-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName -name $NodeTypeName -PlacementProperty @{NodeColor="Red";SomeProperty="6";} -Verbose
```

Düğüm türünün yerleşim özellikleri Bu, varsa eski yerleşim özellikleri üzerine yazılır.

### Örnek 3
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Set-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName  -Name $NodeTypeName -Reimage -NodeName nt1_0, nt1_3
```

Düğüm türündeki ReImage node 0 ve 3.

### Örnek 4
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
$nodeType = Get-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName -Name $NodeTypeName

$nodeType.VmInstanceCount = 6
$nodeType | Set-AzServiceFabricManagedNodeType
```

Boru ile düğüm türünün örnek sayımını güncelleyin.
