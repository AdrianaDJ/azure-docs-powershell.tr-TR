## Tanım
**Set-AzPolicyAssignment** cmdlet 'i bir ilke atamasını değiştirir.
KIMLIĞE göre veya ad ve kapsamla bir ödev belirtin.

## ÖRNEKLERDEN

### Örnek 1: görünen adı güncelleştirme
```
PS C:\> $ResourceGroup = Get-AzResourceGroup -Name 'ResourceGroup11'
PS C:\> $PolicyAssignment = Get-AzPolicyAssignment -Name 'PolicyAssignment' -Scope $ResourceGroup.ResourceId
PS C:\> Set-AzPolicyAssignment -Id $PolicyAssignment.ResourceId -DisplayName 'Do not allow VM creation'
```

İlk komut, Get-AzResourceGroup cmdlet 'ini kullanarak ResourceGroup11 adlı bir kaynak grubu alır.
Komut bu nesneyi $ResourceGroup değişkeninde depolar.
İkinci komut, Get-AzPolicyAssignment cmdlet 'ini kullanarak PolicyAssignment adındaki ilke atamasını alır.
Komut bu nesneyi $PolicyAssignment değişkeninde depolar.
Son komutu, $ResourceGroup 'un **RESOURCEID** özelliğiyle tanımlanan kaynak grubundaki ilke atamasında görünen adı güncelleştirir.

### Örnek 2: ilke atamalarına yönetilen kimlik ekleme
```
PS C:\> $PolicyAssignment = Get-AzPolicyAssignment -Name 'PolicyAssignment'
PS C:\> Set-AzPolicyAssignment -Id $PolicyAssignment.ResourceId -AssignIdentity -Location 'westus'
```

İlk komut, Get-AzPolicyAssignment cmdlet 'ini kullanarak geçerli abonelikteki PolicyAssignment adındaki ilke atamasını alır.
Komut bu nesneyi $PolicyAssignment değişkeninde depolar.
Son komutu, ilke atamalarına yönetilen bir kimlik atar.

### Örnek 3: yeni ilke parametresi nesnesiyle ilke atama parametrelerini güncelleştirme
```
PS C:\> $Locations = Get-AzLocation | where {($_.displayname -like 'france*') -or ($_.displayname -like 'uk*')}
PS C:\> $AllowedLocations = @{'listOfAllowedLocations'=($Locations.location)}
PS C:\> $PolicyAssignment = Get-AzPolicyAssignment -Name 'PolicyAssignment'
PS C:\> Set-AzPolicyAssignment -Id $PolicyAssignment.ResourceId -PolicyParameterObject $AllowedLocations
```

İlk ve ikinci komutlar adları "Fransa" veya "İngiltere" ile başlayan tüm Azure bölgelerini içeren bir nesne oluşturur.
İkinci komut bu nesneyi $AllowedLocations değişkeninde depolar.
Üçüncü komut, ' PolicyAssignment ' adlı ilke atamasını alır komut bu nesneyi $PolicyAssignment değişkeninde depolar.
Son komutu, PolicyAssignment adlı ilke atamasında parametre değerlerini güncelleştirir.

### Örnek 4: ilke atama parametrelerini ilke parametre dosyasıyla güncelleştirme
Yerel çalışma dizininde aşağıdaki içerikle _AllowedLocations.js_ adlı bir dosya oluşturun.

```
{
    "listOfAllowedLocations":  {
      "value": [
        "uksouth",
        "ukwest",
        "francecentral",
        "francesouth"
      ]
    }
}
```

```
PS C:\> Set-AzPolicyAssignment -Name 'PolicyAssignment' -PolicyParameter .\AllowedLocations.json
```

Komut, yerel çalışma dizininden AllowedLocations.jsilke parametre dosyasını kullanarak ' PolicyAssignment ' adlı ilke atamasını güncelleştirir.

### Örnek 5: enforcementMode 'ı güncelleştirme
```
PS C:\> $ResourceGroup = Get-AzResourceGroup -Name 'ResourceGroup11'
PS C:\> $PolicyAssignment = Get-AzPolicyAssignment -Name 'PolicyAssignment' -Scope $ResourceGroup.ResourceId
PS C:\> Set-AzPolicyAssignment -Id $PolicyAssignment.ResourceId -EnforcementMode Default
```

İlk komut, Get-AzResourceGroup cmdlet 'ini kullanarak ResourceGroup11 adlı bir kaynak grubu alır.
Komut bu nesneyi $ResourceGroup değişkeninde depolar.
İkinci komut, Get-AzPolicyAssignment cmdlet 'ini kullanarak PolicyAssignment adındaki ilke atamasını alır.
Komut bu nesneyi $PolicyAssignment değişkeninde depolar.
Son komutu, $ResourceGroup 'un **RESOURCEID** özelliğiyle tanımlanan kaynak grubundaki Ilke atamasında enforcementMode özelliğini güncelleştirir.
