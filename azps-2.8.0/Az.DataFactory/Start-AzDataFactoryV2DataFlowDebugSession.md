# Start-AzDataFactoryV2DataFlowDebugSession

## SYNOPSIS
Azure Veri Fabrikası 'nde veri akışı hata ayıklama oturumu başlatır

## INDEKI

### ByFactoryName (varsayılan)
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob]
 [-ResourceGroupName] <String> [-DataFactoryName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByFactoryObject
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob]
 [-DataFactory] <PSDataFactory> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### Byresourceıd
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob] [-ResourceId] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Tanım
Bu uzun süre çalışan komut yaklaşan hata ayıklama komutları için veri akışı hata ayıklama oturumu başlatır. Bu komut, hata ayıklama oturumu kümesinin boyutunu/türünü yapılandırmak için Integration Runtime tanımıyla birlikte iliştirebilirsiniz.
Veri akışı hata ayıklama iş akışı için PowerShell komut dizisi:
1. Start-AzDataFactoryV2DataFlowDebugSession
1. Add-AzDataFactoryV2DataFlowDebugSessionPackage
1. Invoke-AzDataFactoryV2DataFlowDebugSessionCommand (Bu adımı farklı komutlar/hedefler için tekrarlama veya paket dosyasını değiştirmek için 2-3 adımını tekrarlama)
1. Stop-AzDataFactoryV2DataFlowDebugSession
