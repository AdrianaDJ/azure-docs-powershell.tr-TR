# Remove-AzBatchTask

## SYNOPSIS
Toplu Iş görevini siler.

## INDEKI

### Kimliğe
```
Remove-AzBatchTask [-JobId] <String> [-Id] <String> [-Force] -BatchContext <BatchAccountContext>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObject
```
Remove-AzBatchTask [-InputObject] <PSCloudTask> [-Force] -BatchContext <BatchAccountContext>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Tanım
**Remove-AzBatchTask** cmdlet 'ı bir Azure toplu görevini siler.
*Force* parametresini belirtmediğiniz sürece, bu cmdlet onaylamanızı ister.

## ÖRNEKLERDEN

### Örnek 1: KIMLIĞE göre toplu görev silme
```
PS C:\>Remove-AzBatchTask -JobId "Job-000001" -Id "Task23" -BatchContext $Context
```

Bu komut, kimliği Iş-000001 olan işin altında KIMLIK Task23 olan bir görevi siler.
Komut sizden onaylamanızı ister.
$Context değişkenine bağlam atamak için **Get-AzBatchAccountKeys** cmdlet 'ini kullanın.

### Örnek 2: onay olmadan ardışık düzeni kullanarak toplu görev silme
```
PS C:\>Get-AzBatchTask -JobId "Job-000001" -Id "Task26" -BatchContext $Context | Remove-AzBatchTask -Force -BatchContext $Context
```

Bu komut, **Get-AzBatchTask** cmdlet 'ini kullanarak 000001 Job-olan Iş 'Te kimliği Task26 olan toplu iş görevini alır.
Komut, ardışık düzen işlecini kullanarak bu görevi geçerli cmdlet 'e geçirir.
Komut bu görevi siler.
Bu komut *Force* parametresini belirtir.
Bu nedenle, komut sizden onay istemez.

## PARAMETRELERINE

### -BatchContext
Bu cmdlet 'in toplu Iş hizmetiyle etkileşimli çalışmak için kullandığı **Batchaccountcontext** örneğini belirtir.
BatchAccountContext 'i almak için Get-AzBatchAccount cmdlet 'ini kullanıyorsanız, toplu Iş hizmetiyle etkileşim kurarken Azure Active Directory kimlik doğrulaması kullanılır. Bunun yerine paylaşılan anahtar kimlik doğrulamasını kullanmak için, Get-AzBatchAccountKeys cmdlet 'ini kullanarak erişim anahtarlarının doldurulduğuna bir BatchAccountContext nesnesi alın. Paylaşılan anahtar kimlik doğrulaması kullanırken, birincil erişim anahtarı varsayılan olarak kullanılır. Kullanılacak anahtarı değiştirmek için BatchAccountContext. KeyInUse özelliğini ayarlayın.
