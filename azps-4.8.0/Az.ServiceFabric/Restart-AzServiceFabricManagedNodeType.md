# Restart-AzServiceFabricManagedNodeType

## SYNOPSIS
Düğüm türünden belirli düğümleri yeniden başlatın.

## INDEKI

```
Restart-AzServiceFabricManagedNodeType [-ResourceGroupName] <String> [-ClusterName] <String> [-Name] <String>
 -NodeName <String[]> [-ForceRestart] [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## Tanım
Düğüm türünden belirli düğümleri yeniden başlatın. Hizmet yapısı düğümlerini, VM 'leri yeniden başlatmadan önce devre dışı bırakmış ve geri geldikten sonra tekrar etkinleştirmeyecektir. Bu, birincil düğüm türlerinde yapıldığında, aynı anda tüm düğümleri yeniden başlatmayabilir. Use-ForceRestart işlemi, hizmet yapısı düğümleri devre dışı bırakmasa da işlemi zorunlu tutar, ancak bu işlem sonucunda durum bilgisi olan iş yükleri çalışırken veri kaybına neden olabileceğinden dikkatli kullanın.

## ÖRNEKLERDEN

### Örnek 1
```powershell
$rgName = "testRG"
$clusterName = "testCluster"
$NodeTypeName = "nt1"
Restart-AzServiceFabricManagedNodeType -ResourceGroupName $rgName -ClusterName $clusterName  -Name $NodeTypeName -NodeName nt1_0, nt1_3
```

Düğüm türünde 0 ve 3 düğümünü yeniden başlatın.
