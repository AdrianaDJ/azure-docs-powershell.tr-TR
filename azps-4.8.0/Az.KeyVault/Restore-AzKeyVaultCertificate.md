# Restore-AzKeyVaultCertificate

## SYNOPSIS
Bir yedekleme dosyasındaki anahtar kasasındaki sertifikayı geri yükler.

## INDEKI

### ByVaultName (varsayılan)
```
Restore-AzKeyVaultCertificate [-VaultName] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Restore-AzKeyVaultCertificate [-InputObject] <PSKeyVault> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### Byresourceıd
```
Restore-AzKeyVaultCertificate [-ResourceId] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Tanım
**Restore-Azanahtarvaultcertificate** cmdlet 'i, bir yedekleme dosyasından belirtilen anahtar kasasına sertifika oluşturur.
Bu sertifika, giriş dosyasındaki yedeklenen sertifikanın bir yinelemesidir ve özgün sertifikayla aynı adı vardır.
Anahtar Kasası aynı adla bir sertifika içeriyorsa, özgün sertifikanın üzerine yazılması yerine bu cmdlet başarısız olur.
Yedeklemede sertifikanın birden çok sürümü varsa, tüm sürümler geri yüklenir.
Sertifikayı geri yüklediğiniz tuş Kasası, sertifikayı yedeklediğiniz anahtar kasasından farklı olabilir.
Ancak, Anahtar Kasası aynı aboneliği kullanmalıdır ve aynı Coğrafya (örneğin Kuzey Amerika) bir Azure bölgesinde olmalıdır.
https://azure.microsoft.com/support/trust-center/)Azure bölgelerinin coğrafi olarak eşleştirilmesi Için Microsoft Azure Güven Merkezi 'ne bakın.
